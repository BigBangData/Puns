This simple website provides a question-and-answer game for guessing puns. Signup and login to start playing.

User is given a question and must attempt to answer it and use a pun. If the answer given is close to the spirit of the actual answer, a success message appears, otherwise, a failure message appears. The user must click a button to agree or disagree with the message to load a new question.

The user views a table of models which compared the user answer with the actual answer and scored their similarity. This table contains the models' current match scores for the evaluated answers.

Technically, it is a weighted average of all models' scores that is evaluated against a threshold, and if above the threshold, the user's answer is deemed successful. When the user provides feedback on the website's reaction, model weights are updated accordingly, to penalize the models that got it wrong and promote the models that got it right and in time generate a more accurate weighted average. Since there are positive (AGREE) and negative (DISAGREE) forms of feedback and the website's reaction can also be positive (success message) or negative (failure message), this gets a wee bit complex but not too much. This means that the users "train" the system by answering and providing feedback. If users train the system in bad faith and provide wrong feedback, the system will not improve.
